# DR Bugs

Purpose: record bugs discovered in the original GS/GSL behavior when DireEngine work must correct them to achieve working runtime behavior.

Usage:

- add one entry per confirmed original GS/GSL bug that required a local corrective fix
- include the relevant script ID or canon source when known
- describe the original bug, the observed consequence, and the DireEngine-side correction
- do not log ordinary DireEngine implementation bugs here; this file is only for defects traced back to original canon logic or source data

## YYYY-MM-DD - Short title
