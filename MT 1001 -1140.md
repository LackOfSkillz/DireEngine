Required Fix Pass Before Implementation

These corrections are mandatory before building against this plan:

1. Godot networking examples must use Godot 4 APIs.
2. Local map extraction must traverse exits, not room contents.
3. Map payload generation must tolerate rooms without map coordinates.
4. Initial state delivery must be sent from a real login/puppet hook.
5. Structured websocket payload format must be validated with a ping test before further client work.
6. Client-to-server command format must be validated before any map or UI implementation begins.

Phase 0 — Transport Validation Gate

Do not proceed past MT1005 until both contracts are verified end to end.

Contract 1: Server -> Client

- Godot receives the test payload.
- JSON parses cleanly.
- The canonical shape is confirmed and documented.

Contract 2: Client -> Server

- Godot sends a test command.
- Evennia executes it.
- The response returns to the client.

Implementation note:

The Evennia Godot websocket contrib uses the same JSON-array envelope in both directions. Client commands should be sent in the form:

["text", ["look"], {}]

Do not assume that plain text frames like "look" will be accepted by the contrib.

If either contract fails, stop and adapt the server helper or Godot parser before continuing.

Browser Delivery Pivot

Implementation is now targeting browser-native delivery through Evennia's existing webclient route rather than a required native Godot install.

This changes execution as follows:

1. Server-side transport, structured payloads, map extraction, and character payload work remain valid.

3. UI, map rendering, click-to-move, inventory, hotbar, chat, persistence, and polish are now being implemented in the browser webclient override under web/templates/webclient and web/static/webclient.
4. Godot artifacts may still exist as prototype references, but browser delivery is the active target for player-facing use.

🧱 MT1006 — Create local map extractor
File:
world/area_forge/map_api.py

Add:

def get_local_map(character, radius=3):
    origin = character.location
    if not origin:
        return {"rooms": [], "edges": [], "player_room_id": None}

    visited = set()
    queue = [(origin, 0)]
    rooms = []
    edges = []

    origin_x = getattr(origin.db, "map_x", 0)
    origin_y = getattr(origin.db, "map_y", 0)

    while queue:
        room, dist = queue.pop(0)
        if not room or room.id in visited or dist > radius:
            continue

        visited.add(room.id)

        room_x = getattr(room.db, "map_x", 0)
        room_y = getattr(room.db, "map_y", 0)

        rooms.append({
            "id": room.id,
            "x": room_x - origin_x,
            "y": room_y - origin_y,
            "name": room.key,
            "is_player": room.id == origin.id,
        })

        for exit_obj in room.exits:
            dest = exit_obj.destination
            if not dest:
                continue

            edges.append({
                "from": room.id,
                "to": dest.id,
                "dir": exit_obj.key,
            })

            queue.append((dest, dist + 1))

    return {
        "rooms": rooms,
        "edges": edges,
        "player_room_id": origin.id,
    }

Validation:

Returns nearby rooms
Player position included
Traversal is based on exits, not contents
Gracefully handles rooms without coordinates

Implementation note:

Traversal must not depend on map coordinates. Coordinate presence only controls how a discovered room is rendered in the client payload.

var save_path = "user://settings.json"

func save_settings(data):
    var file = FileAccess.open(save_path, FileAccess.WRITE)
    file.store_string(JSON.stringify(data))
    file.close()

func load_settings():
    if not FileAccess.file_exists(save_path):
        return {}

    var file = FileAccess.open(save_path, FileAccess.READ)
    var content = file.get_as_text()
    file.close()

    return JSON.parse_string(content)
