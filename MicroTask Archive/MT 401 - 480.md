
DragonRealms Stealth System — Practical Overview
Core Philosophy

Stealth in DR is not a toggle.
It is a continuous contested state between stealth and perception.

You are never simply “hidden” — you are harder or easier to detect over time.

🔁 The Core Loop

Everything revolves around this:

Stealth (you) vs Perception (others)

And that contest happens:

when you hide
when you move
when you act
periodically over time
🧱 The Stealth Chain (VERY important)

DR stealth is built as a progression of states:

hide → sneak → stalk → ambush

Each step:

requires the previous
increases control and advantage
increases risk
1. HIDE
Initial concealment attempt
Contested against all observers
Result determines hidden strength

Outcomes:

fail → fully visible
partial → weak concealment
success → normal concealment
strong → very hard to detect

👉 This is where stealth begins

2. SNEAK
Movement while hidden
Every movement triggers new detection checks

Important:

You are not safely hidden
You are continuously tested

Observers may:

see nothing
notice “something”
fully spot you
3. STALK
Target-specific stealth
You attach yourself to a target

Benefits:

stronger concealment vs that target
persistent tracking

Risk:

if target becomes aware, you lose advantage
4. AMBUSH
Stealth attack opener
Requires:
hidden
usually stalking or positioning

Effects:

high accuracy
high damage
bypass or reduce defenses
applies surprise

After:

stealth is broken
⚠️ Detection is NOT binary

Observers do NOT just:

see / don’t see

They experience levels:

nothing → something → movement → exact identity

This creates tension:

“I think something is here”
“I know someone is here”
“There they are”
🧠 Awareness System

Players/NPCs exist in awareness states:

unaware → normal → alert → searching

Effects:

unaware → easy to ambush
alert → harder to hide from
searching → actively scanning

Awareness:

increases after being attacked
decays over time
⚔️ Stealth Combat (Critical Behavior)
Surprise

If target is unaware:

first attack has major advantage
may bypass defenses entirely
Engagement Delay

Target:

does not fully defend immediately
“catches up” after first hit
After First Hit
surprise ends
normal combat resumes
🔄 Stealth is fragile

Stealth breaks when:

you attack
you fail a detection check
you are directly revealed
sometimes from movement
📈 Learning Model

Stealth improves through:

successful hides
moving while hidden
stalking
ambush attempts

But:

trivial attempts don’t teach
impossible attempts teach poorly

def execute(self, user, target=None):
    observers = user.get_room_observers()

    if not observers:
        user.set_state("hidden", {
            "strength": 50,
            "source": "hide"
        })
        user.msg("You slip into hiding.")
        return

    best_diff = None
    best_outcome = "success"

    for observer in observers:
        result = run_contest(
            user.get_stealth_total(),
            observer.get_perception_total()
        )

        if best_diff is None or result["diff"] < best_diff:
            best_diff = result["diff"]
            best_outcome = result["outcome"]

    if best_outcome == "fail":
        user.msg("You fail to find concealment.")
        return

    strength_map = {
        "partial": 10,
        "success": 25,
        "strong": 40
    }

    user.set_state("hidden", {
        "strength": strength_map.get(best_outcome, 10),
        "source": "hide"
    })

    user.msg("You slip into hiding.")
