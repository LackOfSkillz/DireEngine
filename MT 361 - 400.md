note:
I dug into DR’s structure, and the biggest planning takeaway is this:

DragonRealms is organized much more cleanly as skillsets + guild access + spell/ability access than as one giant flat list. The five global skillsets are Armor, Weapons, Magic, Survival, and Lore, and guilds differ mainly by how fast they learn those skillsets and what guild-only systems they can access.

The other very important result: DR does not really have race-based mechanical ability trees in the way many RPGs do. Race affects attributes and mostly RP flavor, and the general new player guide explicitly says there are no racial traits like infravision or racial abilities like faerie fire. So for your design, the clean DR-faithful categorization is:

General skills/abilities: everyone can potentially access these through training or common systems.
Guild/class skills/abilities: learned fastest by some guilds, and many special abilities or spells are guild-specific.
Race modifiers: mostly attribute tendencies, appearance, and flavor verbs rather than a real race-ability tree.

For the skills themselves, DR’s global structure is broad but manageable:

Guild-only skills include at least things like Empathy (Empath), Astrology (Moon Mage), and Expertise (Barbarian). The skills category explicitly marks these as guild-specific and says they can only be learned and taught by members.
The Magic skillset includes Augmentation, Warding, Debilitation, Targeted Magic, Utility, and Sorcery. DR’s magic docs also note that most guilds except Barbarians and Thieves teach spells, while Barbarians and Thieves still interact with the magic/supernatural skillset differently.
The Weapon side also includes umbrella/mastery concepts like Melee Mastery and Missile Mastery, which DR uses as cross-weapon bonus skills.
Tactics is a lore skill with real combat use, including multi-opponent penalties and non-damaging brawling maneuvers.

For guild/class structure, DR currently has 12 guilds/pages in the guild category: Barbarian, Bard, Cleric, Commoner, Empath, Moon Mage, Necromancer, Paladin, Ranger, Thief, Trader, Warrior Mage. Skillset placement determines learning speed and access flavor, not just class identity.

For spells/abilities, the right DR-style mental model is:

General combat and world verbs are available broadly.
Stealth-family verbs should be general or near-general systems gated by skill, not hidden behind a rigid class wall.
Spells and supernatural abilities are strongly guild-linked.
Some guilds should never realistically get certain access paths. Your example is right: a thief should not just pick up Fire Ball, and DR itself ties direct-damage spellcasting to spell-category skill and guild spell access.

So the correct implementation plan for your game is not “dump all skills and abilities on day one.” It should be:

Define a master registry
Every skill and ability exists in data, but each entry has:
category
visibility rule
learnability rule
prerequisite skills
prerequisite guild/class if any
prerequisite item/weapon/stance if any

Split the registry into three top-level groups

General
Guild/Class
Race/Ancestry modifiers

And for DR fidelity, race should mostly modify stats and flavor, not unlock giant ability trees.

Separate skills from abilities
Skills: long-term ranks, trained by doing
Abilities: verbs/actions/spells/maneuvers that consume state and depend on skills
Do not show inaccessible abilities
A new player should only see:
things everyone can try now
things their guild can eventually access
things they have already discovered/learned
maybe “unknown” placeholders only if you want mystery
Use progressive reveal
This is the big one. A player should not see “Fireball” unless they are in a magic path that can learn it. A thief might see hide, stalk, ambush, pick, disarm trap, but not fire magic. That is DR-consistent in spirit.

My recommendation for your actual rollout is:

Phase A: global skill registry
Phase B: active ability framework
Phase C: general abilities everyone can access
Phase D: stealth tree
Phase E: guild spell/ability trees
Phase F: race flavor and stat modifiers, not big race powers

And I would categorize your first implementation wave like this:

General skills/abilities

combat basics
first aid / tending
perception / searching
stealth foundations like hide, sneak, stalk
movement/range verbs
tactics
common interaction verbs

Guild/class skills/abilities

magic spellbooks by guild
barbarian roars/forms/expertise-like systems
thief-specific advanced stealth/crime tools
paladin/cleric holy systems
empath healing/manipulation systems
moon mage astrology/prediction/astral systems
ranger nature/bow/tracking affinities
trader commerce systems

Race-related

attribute aptitudes
appearance variants
flavor verbs
maybe a very light optional ancestry perk layer if you want, but that would be more your invention than DR’s. DR itself does not emphasize racial mechanics the way many RPGs do.
